# Capstone 루브릭

## 목적

이 루브릭은 capstone 산출물이 단순히 길거나 그럴듯한 글인지, 실제로 AI 코딩 하네스 설계 능력을 보여주는지 평가합니다.

## 평가 기준

| 기준 | 통과 | 보완 필요 | 실패 |
|------|------|-----------|------|
| 분석 정확성 | 5개 비교 차원을 모두 다루고 각 판단이 기존 근거 링크로 뒷받침된다. | 일부 차원이나 근거가 약하다. | 차원 구분이 무너지거나 근거 없는 평가가 많다. |
| 설계 일관성 | 목표, non-goals, 도구 표면, 루프, prompt, architecture, model routing, verification이 서로 맞물린다. | 일부 결정이 목표와 약하게 연결된다. | 결정들이 서로 충돌하거나 목표와 무관하다. |
| 근거 링크 | 각 핵심 결정에 기존 분석, framework, 비교 문서, 챕터 링크가 있다. | 링크가 문서 단위로는 있으나 결정 단위 추적성이 약하다. | 근거 링크가 거의 없거나 장식적으로만 붙어 있다. |
| 구현 가능성 | toy 구현 계획이 작고, 파일 구조와 검증 시나리오가 실행 가능하다. | 범위가 조금 크거나 실패 시나리오가 약하다. | 완제품 수준으로 부풀었거나 검증할 수 없다. |

## 통과 기준

- 네 기준 모두 “통과” 또는 하나 이하의 “보완 필요”이어야 합니다.
- “실패”가 하나라도 있으면 capstone은 통과하지 않습니다.
- 보완 필요가 있는 경우 산출물에 수정 계획을 남겨야 합니다.

## 체크리스트

### 분석 정확성

- [ ] 5개 비교 차원을 모두 사용했다.
- [ ] 가치중립 명사구로 접근을 요약했다.
- [ ] 각 차원에 기존 분석 또는 source 근거가 있다.
- [ ] 추천·순위·별점 표현을 피했다.

### 설계 일관성

- [ ] 목표와 non-goals가 분리되어 있다.
- [ ] 도구 표면이 목표에 필요한 만큼만 열려 있다.
- [ ] 루프 종료 조건과 검증 계획이 일치한다.
- [ ] 모델 라우팅/fallback이 품질 한계를 숨기지 않는다.

### 근거 링크

- [ ] 각 decision driver에 근거 링크가 있다.
- [ ] 챕터 링크와 기존 분석 링크가 함께 있다.
- [ ] 링크 없는 판단은 명시적 가정으로 표시했다.
- [ ] final 문서에서 evidence chain을 따라갈 수 있다.

### 구현 가능성

- [ ] toy 구현 계획이 하나의 핵심 decision만 검증한다.
- [ ] 파일 구조가 작고 설명 가능하다.
- [ ] 정상 경로와 실패 경로가 모두 있다.
- [ ] 검증 시나리오가 실제 입력과 기대 동작을 가진다.

## 채점 메모 양식

```text
근거 링크: 통과 / 보완 필요 / 실패
구현 가능성: 통과 / 보완 필요 / 실패

가장 강한 점:
가장 약한 점:
수정해야 할 blocker:
다음 revision에서 확인할 증거:
```
