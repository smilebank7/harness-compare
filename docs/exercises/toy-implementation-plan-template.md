# Toy 구현 계획 템플릿

## 목적

이 템플릿은 capstone 3단계 산출물입니다. 설계안을 완제품 구현으로 키우지 않고, 하나의 학습 가능한 toy 범위로 줄여 검증 가능한 구현 계획을 만듭니다.

## 사용 방법

1. [설계안 템플릿](design-proposal-template.md)에서 가장 중요한 decision driver 하나를 고릅니다.
2. 그 결정을 검증할 최소 기능만 남깁니다.
3. 파일 구조, 구현 단계, 검증 시나리오, 실패 처리를 적습니다.
4. 실제 구현 전에 [루브릭](rubric.md)으로 범위가 너무 큰지 확인합니다.

### 2. 파일 구조

```text

├─ README.md
├─ src/
│  ├─ cli.ts
│  ├─ loop.ts
│  ├─ tools.ts
│  └─ prompts.ts
└─ tests/
   └─ loop.test.ts
```

### 3. 구현 단계

1. CLI 입력과 config를 정의한다.
2. 도구 표면을 최소 1~2개로 제한한다.
3. 단일 루프와 종료 조건을 구현한다.
4. prompt contract를 문서화한다.
5. 검증 시나리오를 작성한다.

## 체크리스트

- [ ] 하나의 핵심 결정만 검증한다.
- [ ] 파일 구조가 5개 이하의 핵심 파일로 설명된다.
- [ ] 정상 경로와 실패 경로가 모두 있다.
- [ ] 검증 시나리오가 실행 가능한 입력/기대 동작을 갖는다.
- [ ] toy 범위가 설계안 전체 구현으로 부풀지 않았다.
